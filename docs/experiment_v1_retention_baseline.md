# 📘 Experiment Record – 2025 Sep–Oct Retention Experiment

**Version:** v1.0

---

# 1. 실험명

**Dandani Retention Experiment v1 — Daily Challenge 기반 사용자 리텐션 검증 (2025.09 ~ 2025.10)**

---

# 2. 실험 목적

Daily Challenge(오늘의 실천) 기반 구조가
“사용자를 매일 돌아오게 만드는 루틴(습관)”을 만들 수 있는지 검증한다.

즉, 단단이의 핵심 PM 가설:

> “작은 실천 → 기록 → 감정 변화”의 반복 루프가
> 자연스러운 재방문을 만들어줄 것이다.

---

# 3. 실험 기간

* **2025.09.23 ~ 2025.10.08**
* 총 16일간의 실사용 및 지표 수집

---

# 4. 실험 배경 및 트래픽 유입 구조

## 4.1 유저 유입 방식

1. **인스타그램 CPC 광고 (9/24 단일 집행)**

* 총 340회 노출
* 37회 클릭
* 클릭당 113원
* 타겟팅: 40~50대 여성 중심 (98% 여성)

2. 자연 유입 (지인 공유, DM, 링크 클릭 포함)

이 기간 동안 온보딩 완료는 0건이며,
Daily Challenge 바로 보기 → 실천 완료 구조로 유입됨.

---

# 5. 실험 환경(기술 스냅샷)

### (이미 공유한 Snapshot 기반, 간단 요약)

* Front: React + Material-UI + Emotion
* Back: Cloudflare Workers + D1
* 익명 사용자 기반 (UUID)
* 실천 기능, 달력 UI, AI 상담 제공
* Daily Report Cron 운영 중
* Notification 없음
* 온보딩은 모달 기반 하나뿐

---

# 6. 실험 변수 (독립 변수)

단단이 v1의 Daily Loop 구조:

**① 새로운 실천을 확인한다**
**② 실천을 한다**
**③ 기록을 남긴다**
**④ 피드백을 남긴다**

피드백은 실험 기간 중 거의 모든 실천이 발생한 날 함께 전송됨.

---

# 7. 종속 변수 (실험 지표)

실험 기간에 측정한 지표:

* Day1 Retention
* Day7 Retention
* Week1 Completion
* Day30 Completion
* Positive Feedback 비율
* DAU(일간 활성 사용자)

* AI 상담 이용률: 0%

# 9. 원인 분석 (FA Analysis)

### 9.1 UX 구조 문제

* Daily Challenge가 “무한 스크롤 같은 구조”
  → 목표가 없음
* 실천 완료 후의 변화/보상 없음
* 다음 날 재방문을 유도하는 Hook 부재

### 9.2 기능적 문제

* 알림/리마인더 없음
* 온보딩이 단 1번 모달
* AI 상담은 존재하나 노출 비중 매우 낮음

### 9.3 사용자 층 특성

인스타 타겟팅(40~50대 여성)은
**1일 루틴 정착까지 매우 구체적 동기 유발이 필요**
그러나 단단이 v1은

* 내 상태를 기록해주는 기능
* 루틴화된 챌린지 구조
  이 둘 모두 부족함.

### 9.4 심리적 Hook 부족

감정 회복력이라는 추상적 목표가
행동 개입(Behavioral Activation)으로 이어지기 어려움.

# 10. 커밋 로그 기반의 실제 개발 상태(실험 당시)

### 10.1 개발이 가장 집중된 기간

* **2025-09-23 ~ 2025-10-02**

이 기간 동안 핵심 기능 대부분이 완성되었고,
실험이 실질적으로 이 MVP를 기반으로 진행됨.

### 10.2 버전 이력

* v0.14.0 — 40~50대 여성 UX 개선
* v0.15.0 — 폰트/디자인 리뉴얼
* v0.16.0 — 실천 달력 기능 추가

### 10.3 개발 패턴

* 주로 `feat:` 중심의 기능 개발
* Bugfix 일부
* Conventional Commits 준수

### 10.4 실험 당시 완료되지 않은 작업

* 리텐션 지표 계산 완성 전 (미커밋 부분 있음)
* 새 스키마 반영 전
* 온보딩 고도화 미완

➡ **즉, 완성도 70~75%의 MVP 상태에서 실험이 진행됨.**

### 10.5 작업 중단 기간

* 2025-10-02 이후 약 7주간 개발 중단
  → 실험 개선 루프가 이어지지 못함
  → 리텐션 개선을 위한 기능 실험이 이루어지지 못함

# 11. 결론

이번 실험은 **Daily Challenge 구조만으로는 리텐션을 만들 수 없다**는 것을 명확히 보여준다.

또한 **단단이의 코어 Loop는 아직 정의되지 않았음**이 드러났다.

### 정리된 결론

| 항목                    | 결론              |
| --------------------- | --------------- |
| Daily Challenge 기반 루프 | ❌ 유효하지 않음       |
| 익명 사용자 기반             | 재방문 유도에 취약      |
| 하루 1실천 구조             | 동기 부여 부족        |
| UX                    | 재방문 설계 부족       |
| AI 상담 기능              | 사실상 무의미 (0% 사용) |

가장 중요한 결론은 다음 문장으로 요약할 수 있다:

> **단단이 v1은 “루틴 생성 서비스”가 아니라 “1일 체험 서비스”였다. 라이프사이클을 만들기 위한 장치가 없었다.**
