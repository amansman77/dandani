# ADR-PROD-002 Navigation Structure — From Challenge Tabs to Action-Centric UI

## 상태
제안

---

## 맥락

Dandani V2는 다음과 같이 재정의되었다:

> 개인의 감정 상태와 행동 반응을 기반으로  
> 지금 실행 가능한 작은 행동을 제안하는 개인화 에이전트

하지만 기존 V1의 네비게이션 구조는 다음과 같았다:

- 챌린지
- 챌린지 도우미
- 내 기록

이 구조는 다음과 같은 문제를 가진다:

- 사용자가 "행동"이 아니라 "챌린지를 선택하는 앱"으로 인식한다
- 콘텐츠 탐색 구조가 행동 실행을 지연시킨다
- 에이전트 중심 UX와 충돌한다
- 기능 간 역할이 중복된다 (챌린지 도우미 vs 단단이)

따라서 다음 질문이 발생했다:

> "단단이에서 사용자는 어디로 들어가고, 무엇을 하게 되어야 하는가?"

---

## 결정

Dandani V2의 네비게이션은  
**행동 중심 구조(Action-Centric Navigation)**로 재설계한다.

---

### 1. 기본 원칙

- 탐색보다 실행을 우선한다
- 선택보다 제안을 우선한다
- 기능보다 흐름을 중심으로 구성한다

---

### 2. 탭 구조 단순화

기존 3탭 구조를 다음 2탭 구조로 변경한다:

```
[한 걸음]   [기록]
```

---

### 3. 각 탭의 역할

#### (1) 한 걸음

역할:

- 단단이 에이전트와의 상호작용
- 현재 상태 입력
- 행동 제안
- 행동 실행
- 결과 입력

즉:

> "지금 나를 위한 행동을 수행하는 공간"

---

#### (2) 기록

역할:

- 과거 행동 기록 조회
- 결과 및 느낌 확인
- 패턴 인식

즉:

> "내가 어떻게 행동해왔는지 보는 공간"

---

### 4. 제거 / 통합 대상

#### 챌린지 탭

- 기존 "선택형 챌린지 리스트" 제거
- 행동은 항상 에이전트가 제안

---

#### 챌린지 도우미 탭

- 기능 제거
- 단단이 에이전트로 통합

---

## 근거

### 1. 선택은 실행을 지연시킨다

- 리스트 기반 선택은 인지 부하를 증가시킨다
- 행동 시작까지의 시간이 길어진다

---

### 2. 에이전트 중심 구조와 충돌

- 에이전트는 "추천"이 아니라 "제안"한다
- 탭 기반 탐색은 에이전트 흐름을 깨뜨린다

---

### 3. 단일 행동 구조와의 정합성

- Dandani는 한 번에 하나의 행동만 제안한다
- 다중 선택 UI는 구조적으로 맞지 않는다

---

### 4. 기능 중복 제거

- "챌린지 도우미"는 단단이와 역할이 동일
- 분리 시 혼란 발생

---

### 5. 집중도 향상

- 탭이 줄어들수록 사용자의 행동 경로가 명확해진다
- 행동 실행까지의 거리가 짧아진다

---

## 대안 비교

### 대안 1: 기존 3탭 유지

- 장점: 기존 사용자 익숙함
- 단점: 행동 중심 구조와 충돌 → 기각

---

### 대안 2: 3탭 유지 + 의미 변경

- 장점: 점진적 변화
- 단점: 구조 혼재 → 기각

---

### 대안 3: 단일 화면 (탭 없음)

- 장점: 극단적 단순화
- 단점: 기록 접근성 저하 → 보류
