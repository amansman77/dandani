# ADR-PROD-000 Dandani V2 — Action Personalization Agent

## 상태
제안

---

## 맥락

기존 단단이 V1은 "오늘의 챌린지" 형태로,  
서버가 공통된 행동을 사용자에게 제공하는 구조였다.

이 구조는 다음과 같은 한계를 가진다:

- 사용자의 현재 감정 상태를 반영하지 못함
- 동일한 행동이 사용자마다 다르게 작용함
- 행동 실패의 원인을 학습하지 못함
- 장기적으로 개인화된 변화 패턴을 만들 수 없음

또한 대부분의 서비스는 다음과 같은 방식으로 개인화를 수행한다:

- 콘텐츠 추천 (영상, 글, 상품 등)
- 소비 기반 만족 최적화

하지만 이는 다음 문제를 가진다:

- 사용자는 더 많이 소비하지만 실제 행동 변화는 없음
- 감정 상태는 변해도 실제 상황은 바뀌지 않음
- 반복 사용 대비 장기적 가치 축적이 약함

이로 인해 단단이는 다음 질문에 답해야 했다:

> "이 사람에게 지금 필요한 것은 무엇을 보는 것이 아니라, 무엇을 하는 것 아닌가?"

---

## 결정

단단이는 다음과 같이 재정의한다:

> 단단이는 개인의 감정 상태와 행동 반응을 기반으로  
지금 실행 가능한 작은 행동을 제안하는 개인화 에이전트다

이를 위해 다음 구조를 채택한다:

### 1. 행동 개인화 (Action Personalization)

- 콘텐츠가 아니라 행동을 개인화한다
- 추천이 아니라 실행을 목표로 한다

---

### 2. 단일 행동 제안

- 한 번에 하나의 행동만 제안한다
- 행동은 반드시 다음 조건을 만족해야 한다:
  - 10분 이내
  - 단일 단계
  - 즉시 실행 가능

---

### 3. 행동 결과 구조

행동 결과는 다음 2단계로 수집한다:

- 시작 여부 (Start)
- 완료 여부 (Sustain)

이를 통해 다음 상태를 구분한다:

- 미시작 (Start ❌)
- 부분 수행 (Start ✅, Sustain ❌)
- 완료 (Start ✅, Sustain ✅)

---

### 4. 패턴 학습 구조

각 세션에서 다음 데이터를 축적한다:

- 감정 상태 (State)
- 제안된 행동 (Action)
- 행동 결과 (Response)
- 수행 후 느낌 (Feeling)

이를 기반으로:

- 행동 시작 패턴
- 행동 유지 패턴

을 학습한다.

---

## 근거

### 1. 행동이 아닌 콘텐츠는 상태를 바꾸지 못한다

- 콘텐츠 소비는 일시적인 감정 변화만 유도한다
- 실제 변화는 행동을 통해서만 발생한다

---

### 2. 행동 실패는 중요한 데이터다

기존 시스템:

- 성공 여부만 기록

단단이:

- 시작 실패
- 유지 실패

모두를 학습 대상으로 본다

---

### 3. 개인화의 대상은 "취향"이 아니라 "실행 가능성"이다

대부분 서비스:

- 무엇을 좋아하는가

단단이:

- 무엇을 할 수 있는가

---

### 4. 대안과 비교

#### 대안 1: 콘텐츠 기반 개인화

- 장점: 구현 쉬움
- 단점: 행동 변화 없음 → 기각

---

#### 대안 2: 다중 선택 행동 제안

- 장점: 사용자 선택권 증가
- 단점: 선택 피로, 실행률 저하 → 기각

---

#### 대안 3: 완전 자동 에이전트

- 장점: 자동화 극대화
- 단점: 사용자 주체성 약화 → 초기 단계에서는 부적합 → 보류
