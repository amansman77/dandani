# ADR-PROD-003 Action Taxonomy — 단단이 행동 타입 시스템

## 상태
제안

---

## 1. 맥락

Dandani V2는 다음을 목표로 한다:

> 사용자 상태 기반으로 "지금 실행 가능한 행동"을 제안하는 개인화 에이전트

하지만 현재 문제:

- 행동이 매번 생성됨 (비일관성)
- UI 패턴이 매번 다름
- 캐릭터 활용이 구조화되지 않음

**핵심 질문:**

> "행동을 어떻게 구조화할 것인가?"

---

## 2. 결정

단단이의 행동을 **Action Type (행동 타입)** 으로 분류한다.

---

## 3. Action Type 구조

모든 행동은 다음 구조를 가진다:

```json
{
  "type": "START | CALM | FOCUS | MOVE | RELEASE | REFLECT",
  "difficulty": "LOW | MID | HIGH",
  "duration": "SHORT | MEDIUM",
  "interaction_mode": "MIRROR | TRIGGER | COMPANION | SCENE",
  "character_state": "단단이 상태",
  "action_payload": "실제 행동 내용"
}
```

---

## 4. 핵심 Action Type 6가지

### 1) START (시작 유도형)

**목적:** 아무것도 못하는 상태 → 시작

**상황:** 미루기 / 무기력 / 압박

**예:**
> 3, 2, 1 — 지금 딱 10초만 시작

**UI:** 카운트다운 / 버튼 최소화

**캐릭터:** 코딩 단단이 / 작업 단단이

---

### 2) CALM (안정형)

**목적:** 감정 진정 / 긴장 완화

**상황:** 피로 / 스트레스 / 불안

**예:**
> 눈 감고 3번만 숨 쉬자

**UI:** 호흡 애니메이션 / 반복 리듬

**캐릭터:** 휴식 단단이 / 책 읽는 단단이

---

### 3) FOCUS (집중형)

**목적:** 주의 집중 / 몰입 시작

**상황:** 산만함 / 집중 안됨

**예:**
> 딱 3분만 이것만 하자

**UI:** 타이머 / 단일 목표 표시

**캐릭터:** 코딩 단단이 / 책상 단단이

---

### 4) MOVE (행동 활성형)

**목적:** 몸 움직이기 / 에너지 회복

**상황:** 무기력 / 답답함

**예:**
> 자리에서 일어나 5걸음만 걸어보자

**UI:** 걸음 카운트 / 진행 애니메이션

**캐릭터:** 등산 단단이 / 산책 단단이

---

### 5) RELEASE (감정 해소형)

**목적:** 감정 배출

**상황:** 화남 / 답답함 / 억눌림

**예:**
> 지금 머릿속에 있는 거 한 줄로 써보자

**UI:** 입력창 / 즉시 기록

**캐릭터:** 일기 단단이

---

### 6) REFLECT (회고형)

**목적:** 의미 정리 / 학습

**상황:** 행동 이후 / 하루 마무리

**예:**
> 오늘 어땠어?

**UI:** 기록 / 요약

**캐릭터:** 밤 단단이 / 캠핑 단단이

---

## 5. Action Type ↔ 캐릭터 연결

| Action Type | 캐릭터 |
|-------------|--------|
| START | 코딩 단단이 |
| CALM | 휴식 / 독서 단단이 |
| FOCUS | 작업 단단이 |
| MOVE | 등산 / 산책 단단이 |
| RELEASE | 일기 단단이 |
| REFLECT | 밤 / 캠핑 단단이 |

**핵심:**
> 캐릭터 = UI / 행동 타입 = 로직

---

## 6. 선택 로직 (핵심)

에이전트는 이렇게 판단한다:

> 상태 → Action Type 선택 → UI 생성

**예:**

- "피곤하고 아무것도 하기 싫어" → CALM → 호흡 UI + 휴식 단단이
- "해야하는데 미루고 있음" → START → 카운트다운 UI + 작업 단단이

---

## 7. 왜 이 구조가 중요한가

### 1) 확장 가능

- 새로운 행동 추가 가능
- 캐릭터 추가 가능

### 2) 일관성 유지

- UX 통일됨
- 사용자 학습됨

### 3) 자동 생성 가능

- LLM → Type만 선택
- UI는 고정

**핵심:**
> "행동 생성"이 아니라 "행동 선택 시스템"

---

## 8. 한 줄 정리

> 단단이는 행동을 만들어내는 게 아니라  
> 상황에 맞는 행동 타입을 선택하는 시스템이다

---

## 9. 지금 상태에서 다음 단계

각 Action Type 별 UI 컴포넌트 정의

| Action Type | UI 컴포넌트 |
|-------------|------------|
| CALM | breathing component |
| START | countdown component |
| MOVE | step counter |
| FOCUS | timer component |
| RELEASE | 입력창 |
| REFLECT | 기록/요약 |
