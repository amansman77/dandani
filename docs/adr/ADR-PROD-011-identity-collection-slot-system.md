# ADR-PROD-011 Identity Collection & Slot System

## 상태
제안

---

## 맥락

Dandani V2는 다음 루프를 중심으로 동작한다:

```
State → Action → Response → Update
```

이를 통해 사용자의 감정과 행동 패턴이 축적된다.

하지만 현재까지의 결과는:

- 행동 제안 ⭕
- 개인화 ⭕
- 패턴 학습 ⭕

👉 그러나 이 모든 것이 흘러가고 사라진다

---

문제:

> ❗ 사용자의 변화가 "보이지 않는다"  
> ❗ 축적된 데이터가 "경험으로 남지 않는다"

---

👉 핵심 질문:

> "사용자의 상태와 변화는 어떻게 보존되고 체감되는가?"

---

## 결정

Dandani는  
사용자의 상태와 행동 패턴을 기반으로 생성된 단단이 캐릭터를 수집 가능한 형태로 저장하고,

이를 통해 사용자의 변화를 시간 기반으로 시각화한다.

---

## 핵심 정의

### Identity Dandani

> 특정 시점의 사용자 상태 + 행동 패턴을 반영하여 생성된 단단이

---

### Collection

> 서로 다른 시점의 단단이들을 모아  
> 사용자의 변화를 기록하는 구조

---

### Slot

> 단단이를 저장할 수 있는 제한된 공간

---

## 1. 시스템 구조

```
User Behavior Data → Pattern Analysis → Identity Dandani Generation → Slot Storage → Collection View
```

---

## 2. Identity Dandani 구성

각 단단이는 다음 정보를 포함한다:

```json
{
  "dandani_id": "uuid",
  "created_at": "2026-05-01",
  "title": "시작하는 단단이",
  "description": "피곤해도 시작은 놓치지 않는 시기의 나",
  "dominant_action": "START",
  "emotion_pattern": "LOW energy, WEAK intent",
  "image_url": "/generated/dandani_123.png"
}
```

---

👉 핵심:

> 단단이는 "이미지"가 아니라  
> 👉 그 시기의 나를 설명하는 기록

---

## 3. 생성 조건

Identity Dandani는 다음 조건에서 생성된다:

- 최소 3~5 cycle 수행
- 패턴이 일정 수준 형성됨
- 사용자가 "나만의 단단이 만들기" 선택

---

👉 이유:

> 데이터 없는 상태에서 생성하면 의미 없음

---

## 4. 생성 UX

```
호성, 지금까지 이런 흐름이 있었어.
 - 피곤해도 시작은 하는 편이야
 - 시작하면 끝까지 가는 힘이 있어

그래서 이런 단단이가 만들어졌어.
```

→ 이미지 생성

---

사용자 선택:

```
[ 이 단단이로 바꿀래 ] [ 나중에 볼게 ]
```

---

## 5. Slot 시스템

### 기본

- 기본 단단이 (항상 존재, 삭제 불가)

---

### 추가 단단이

- 슬롯에 저장
- 슬롯 수 제한 존재

---

```
[ 기본 단단이 ] [ 시작하는 단단이 ] [ 숨 고르는 단단이 ] [ + 슬롯 추가 ]
```

---

## 6. 과금 구조

### 1) 단단이 생성

- 1회 생성 무료 or 제한 제공
- 이후 생성 유료

---

### 2) 슬롯 확장

- 추가 단단이 저장 시 슬롯 필요
- 슬롯 확장 유료

---

👉 핵심:

> 과금은 기능이 아니라 "기록을 남기는 능력"에 대한 비용

---

## 7. UX 원칙

### 1) 수집 = 기록

- 단순 캐릭터 수집 금지
- 반드시 "의미" 포함

---

### 2) 시간성

```
2026.05.01 — 시작하는 단단이
2026.05.10 — 숨 고르는 단단이
```

👉 단단이는 타임라인이 된다

---

### 3) 비교 가능

```
예전 단단이 vs 지금 단단이
```

👉 변화 체감

---

### 4) 이름 부여

```
이 단단이 이름을 지어볼래?
```

👉 소유감 강화

---

## 8. 금지 사항

- 단순 외형 커스터마이징 (색상, 액세서리 등)
- 의미 없는 랜덤 생성
- 패턴과 무관한 캐릭터

---

## 9. 기대 효과

- 리텐션 증가
- 자기 인식 강화
- 행동 패턴의 시각화
- 자연스러운 과금 유도

---

## 10. 확장 가능성

- 단단이 타임라인
- 감정 변화 시각화
- "나의 변화" 리포트
- 회고 기반 콘텐츠

---

## 한 줄 정의

> 단단이는 캐릭터가 아니라  
> 👉 특정 시점의 나를 기록한 형태다

---

## 한 줄 정리

> 단단이를 수집한다는 것은  
> 👉 캐릭터를 모으는 것이 아니라  
> 👉 나의 시간을 모으는 것이다
